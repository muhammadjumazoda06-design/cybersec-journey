## 🔐 OWASP Top 10 (2021)

**OWASP** — организация которая публикует список 10 самых критичных уязвимостей веб-приложений. Стандарт индустрии — обязателен для каждого пентестера!

| # | Название | Описание |
|---|---|---|
| A01 | Broken Access Control | Доступ к чужим данным (IDOR) |
| A02 | Cryptographic Failures | Слабое шифрование, пароли в открытом виде |
| A03 | Injection | SQLi, Command Injection, XSS |
| A04 | Insecure Design | Архитектурные ошибки |
| A05 | Security Misconfiguration | Неправильные настройки сервера |
| A06 | Vulnerable Components | Устаревшие библиотеки |
| A07 | Auth Failures | Слабая аутентификация, брутфорс |
| A08 | Software & Data Integrity | Подделка обновлений |
| A09 | Logging Failures | Нет логов = нет обнаружения атак |
| A10 | SSRF | Сервер делает запросы от своего имени |

> 💡 SQLi, XSS = A03 | Brute Force = A07 | IDOR = A01 | LFI = A05

## 🔍 Nikto — сканер уязвимостей

### Что делает
Автоматически проверяет веб-сервер на известные уязвимости, устаревшее ПО, отсутствующие заголовки безопасности.

### Команда
```bash
nikto -h http://127.0.0.1
```

### Результаты на localhost (DVWA)
```
✅ Найдено 10 проблем за 330 секунд (8083 запросов)

🔴 Критичное:
- /server-status открыт → раскрывает информацию Apache

🟡 Отсутствующие заголовки:
- Content-Security-Policy   → защита от XSS
- Strict-Transport-Security → принудительный HTTPS
- X-Content-Type-Options    → защита от MIME sniffing
- Referrer-Policy           → утечка URL
- Permissions-Policy        → ограничения API браузера

🟡 Прочее:
- ETags → утечка inode (CVE-2003-1418)
- OPTIONS метод открыт
- Apache 2.4.66 (актуальная версия)
```

### Важное правило!
```
⚠️ ВСЕГДА проверяй Target IP перед сканированием!
nikto -h http://127.0.0.1   ← твой сервер ✅
nikto -h http://185.x.x.x   ← чужой сервер ❌ НЕЗАКОННО!
```

## 📁 Gobuster — поиск скрытых директорий

### Что делает
Перебирает словарь и проверяет каждое слово как возможный путь на сервере. Находит скрытые файлы и директории.

### Команда
```bash
gobuster dir -u http://127.0.0.1/dvwa/ -w /usr/share/wordlists/dirb/common.txt
```

### Результаты на DVWA
```
.git/HEAD    200 ← КРИТИЧНО! Git репозиторий открыт!
config/      301 ← папка с конфигами (пароли БД!)
database/    301 ← папка с базой данных
php.ini      200 ← конфиг PHP открыт
phpinfo.php  302 ← полная инфо о PHP сервере
robots.txt   200 ← список скрытых страниц
tests/       301 ← тестовая папка открыта
.htaccess    403 ← заблокировано (хорошо)
```

### Опасность открытого .git
```bash
# Хакер может скачать весь исходный код:
wget http://target.com/.git/HEAD
# → получает структуру репозитория → скачивает весь код включая пароли!
```

### Разница с Nikto
| Инструмент | Что делает |
|---|---|
| **Nikto** | Сканирует известные уязвимости и заголовки |
| **Gobuster** | Брутфорсит пути — находит скрытые файлы |

## 📂 LFI — Local File Inclusion

### Что это
Уязвимость при которой сервер включает локальные файлы через параметр URL. Позволяет читать системные файлы.

### Как работает
```
Нормально:  ?page=file1.php        → показывает нужный файл
Атака:      ?page=/etc/passwd      → читает системный файл!
```

### ../  объяснение
```
Сервер находится в: /var/www/html/dvwa/vulnerabilities/fi/
../  = перейди папку выше
../../../../../../  = выходим в корень /
→ можем читать любой файл системы!
```

### Практика на DVWA
```
# Тест 1 — нормальный файл
?page=file1.php → Hello admin, Your IP: 127.0.0.1

# Тест 2 — системный файл (сработало!)
?page=/etc/passwd → содержимое файла со всеми пользователями!

# Тест 3 — shadow (заблокировано)
?page=/etc/shadow → пусто (Apache не имеет прав)
```

### Что видно в /etc/passwd
```
root:x:0:0:root:/root:/bin/bash        ← суперпользователь
kali:x:1000:1000::/home/kali:/bin/zsh  ← основной пользователь
mysql:x:116:118:MariaDB Server         ← база данных
```

> `/etc/passwd` — пользователи системы (не пароли!)
> `/etc/shadow` — хэши паролей (только root может читать)

### OWASP категория
**A05 — Security Misconfiguration**

## 🔓 IDOR — Insecure Direct Object Reference

### Что это
Приложение не проверяет **кто** делает запрос — только **что** запрашивается. Позволяет получить чужие данные.

### Практика на DVWA
```
# Запрос без авторизации к API:
GET /dvwa/vulnerabilities/authbypass/get_user_data.php

# Ответ сервера (все пользователи без проверки!):
[
  {"user_id":"1","first_name":"admin","surname":"admin"},
  {"user_id":"2","first_name":"Gordon","surname":"Brown"},
  {"user_id":"3","first_name":"Hack","surname":"Me"},
  {"user_id":"4","first_name":"Pablo","surname":"Picasso"},
  {"user_id":"5","first_name":"Bob","surname":"Smith"}
]
```

### Как тестировать IDOR
```
1. Найди параметр с ID: ?id=1
2. Меняй ID: ?id=2, ?id=3, ?id=100
3. Проверяй без авторизации
4. Если сервер возвращает чужие данные → IDOR найден!
```

### Почему опасно
В реальных приложениях там могут быть: email, телефон, адрес, данные карты, медицинские данные!

### OWASP категория
**A01 — Broken Access Control**

### Responsible Disclosure
Если нашёл уязвимость на чужом сайте (даже университета):
- ✅ Сообщи администратору письменно
- ✅ Опиши уязвимость без эксплуатации

### Шаблон письма администратору
```
Тема: Отчёт об уязвимостях сайта [название]

Уважаемый администратор,
Я студент, изучающий кибербезопасность.
Обнаружил следующие проблемы:

1. Открытый листинг директорий (/lib/, /html/)
2. Открытый CKFinder (/ckfinder/)
3. Отсутствуют заголовки безопасности (CSP, HSTS)
4. Cookie без флага HttpOnly
5. Устаревшая версия Apache

Готов предоставить дополнительную информацию.
```

## 🔄 Типичный workflow разведки веб-сайта

```
1. nikto -h http://target     → известные уязвимости
2. gobuster dir -u http://target -w wordlist.txt  → скрытые пути
3. Проверяем найденные пути вручную
4. Тестируем параметры на LFI, SQLi, IDOR
5. Документируем находки
```
